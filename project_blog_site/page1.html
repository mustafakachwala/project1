First Encounter with GIT

Monday, 16 April 2012

Its a kick-start of the Full Stack Web Developer Program and my first encounter with Git.

Well, totally new to this Version Control System (VCS), Git is a Distributed Version Control System. The meaning of Distributive Version Control System is basically that we can create or make changes to files and save it on a local system and then collaborate to the server. Hence, the dependency on server is reduced.

In order to use Git, we need to create an account on www.github.com and install the Git Bash software on the local system. There are a few steps to be followed for the installation which can be found on the following link by Git help.github.com.

One of the important advantages of using Git is that we can collaborate. What I mean by collaboration is that more than one person can work on the same file and all their changes can be merged into that file. In order to collaborate, the user has to give rights to others users. In order to add collaborators, the following steps need to be followed:
  1. Login into your Github account.
  2. Select a particular repository.
  3. Once selected, click on the "Admin" tab, and then click on "Collaborators".
  4. Add the required users to collaborate with. 
There are a number of terminologies associated with Git. The one that I would like to discuss is Git Branching.
Git Branching: Consider you are working on some project, and made some commits. Suppose you decide to work on a particular part, separately. In this case, you can create a branch. In order to create a branch, please follow the below steps:
  1. Open Git Bash. To find out if any branches exit, run the command git branch. The output window below shows master as the only branch, and the "*" indicates and it is the current branch used.
  2. In order to create a branch, run the command git branch < branch-name >
